JWT AUTHENTICATION FLOW (Know This Cold):
================================================================================
1. User submits email + password to /api/auth/login
2. Backend validates with BCrypt
3. If valid, generates JWT token with user details + role
4. Token sent to frontend, stored in localStorage
5. Frontend adds token to Authorization header: "Bearer {token}"
6. JwtAuthenticationFilter validates token on each request
7. If valid, sets SecurityContext with user
8. @PreAuthorize checks role permissions

COMMON INTERVIEW QUESTIONS & ANSWERS:
================================================================================

Q: Why Spring Boot?
A: "Spring Boot provides rapid development with auto-configuration, built-in 
security, easy dependency management, and production-ready features like 
actuator. It's industry-standard for enterprise Java applications."

Q: Why JWT over sessions?
A: "JWT is stateless, making it scalable for microservices and load-balanced 
environments. No server-side session storage needed. Token contains all user 
info, reducing database queries."

Q: Why React?
A: "React's component-based architecture promotes reusability. Virtual DOM 
ensures fast rendering. Large ecosystem with libraries. Easy state management 
for this project size."

Q: How do you prevent SQL injection?
A: "I use Spring Data JPA which uses parameterized queries by default. All 
user inputs are bound as parameters, not concatenated into SQL strings."

Q: How do you handle errors?
A: "I have a global exception handler with @RestControllerAdvice that catches 
all exceptions and returns consistent error responses with proper HTTP status 
codes. Frontend displays user-friendly messages."

Q: What's the difference between @Service and @Repository?
A: "@Service is for business logic layer, @Repository is for data access layer. 
@Repository also provides exception translation for database errors."

Q: Explain One-to-Many relationship.
A: "One Module has Many Questions. In Module entity, I have @OneToMany 
annotation with mappedBy='module'. In Question entity, I have @ManyToOne with 
@JoinColumn pointing to module_id."

Q: How do you test your APIs?
A: "I use Postman for manual testing. I test authentication flow, role-based 
access (403 for wrong roles), CRUD operations, and error scenarios. For 
production, I'd add JUnit tests."

Q: What's the difference between PUT and POST?
A: "POST creates new resources (non-idempotent). PUT updates existing resources 
(idempotent - same request multiple times has same effect). I use POST for 
creating modules, PUT for updating them."

Q: How do you secure passwords?
A: "I use BCrypt hashing with Spring Security's PasswordEncoder. Passwords are 
hashed with salt before storing. Never store plain text. BCrypt is slow by 
design to prevent brute force attacks."
